## Successes and Strengths

### Test Discipline
162 → 319 tests across 11 stories with zero regressions. Every acceptance criterion tested. Proptest added for delta-save roundtrip validation. `test_app()` and `run_until_loaded()` helpers evolved organically.

### Pattern Consistency
Established patterns scaled across all stories without modification:
- `from_ron()` + `Default` fallback (used in 8 configs)
- `#[serde(default)]` for backward-compatible config extension (introduced Story 1.5)
- `NeedsXVisual` marker pattern for core/rendering separation
- Config validation (`validate()` methods from Story 1.6 onward)

### Delta-Save Architecture (Story 1.9)
Seed + changes model keeps save files small. SeedIndex, ChunkDelta, WorldDeltas. Schema versioning with v1→v2 migration from day one. Proptest confirms roundtrip correctness.

### 3-Phase Chunk Loading (Story 1.5)
Refactored `update_chunks` into Unload (immediate) → Queue (Manhattan priority) → Load (staggered). ChunkEntityIndex for O(1) despawn. Extended exploration stays memory-bounded.

### Event System Foundation (Story 1.7)
GameEvent, EventSeverity, Logbook infrastructure. All existing systems retrofitted (collision, weapons, chunks). Future features (logbook UI, telemetry) can subscribe without code changes.

### WASM Build (Story 1.10)
22 MB release build — under 50 MB budget and 30 MB stretch goal. Platform-split Cargo.toml, save system WASM guards, trunk build pipeline.

### Bevy 0.18 Learning Curve
API surprises (MessageWriter.write(), BorderRadius as Node field, Entity::from_bits()) occurred only on first use, then were correctly applied in all subsequent stories.

## Key Insights and Learnings

1. **First-story investment pays compound returns** — Story 1.1's 4 review rounds established entity budget patterns that made all subsequent chunk-related stories smooth. Grinding fundamentals unlocks speed later.

2. **Pattern discipline scales** — Once a pattern is established (config loading, serde defaults, marker components, config validation), it's reused without friction. Document patterns, then follow them.

3. **Config backward compatibility via serde defaults** — `#[serde(default = "fn")]` allows adding fields to RON configs without breaking existing files. Zero migration cost. Use for every new config field.

4. **Bevy API learnings are sticky** — Each API surprise (Messages, UI nodes, entity construction) occurred once then was correctly applied. Keep Dev Notes as institutional memory.

5. **Code reviews catch real bugs** — Entity budget inflation, O(n) performance (Logbook Vec→VecDeque), String allocations in events, missing config validation. Reviews are not ceremony — they're quality infrastructure.
