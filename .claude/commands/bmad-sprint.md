/ralph-loop "You are running an autonomous BMAD sprint loop.

## State Files
- Sprint status: Read from the BMAD sprint-status file
- Review tracking: .bmad-output/review-tracker.yaml

## Pre-Flight
Before each iteration:
1. Read sprint-status to find the next story that is NOT done
2. Read review-tracker.yaml (create it if missing using the template below)

## Workflow Per Story

| Status         | Action                                              |
|----------------|-----------------------------------------------------|
| backlog        | /bmad-gds-create-story → set status ready-for-dev   |
| ready-for-dev  | /bmad-gds-dev-story → set status review             |
| in-progress    | /bmad-gds-dev-story → set status review             |
| review         | Run next review round (see Review Protocol below)   |
| reviewed       | git commit + push → set status done                 |

## Review Protocol

When a story has status 'review':

1. Read review-tracker.yaml for that story
2. Check how many rounds are completed
3. If completed < 3: run the next round with the designated focus
4. If completed >= 3: set sprint-status to 'reviewed'

### Round Focus Areas

Round 1 - Architecture & Spec Conformance:
- Does the implementation match the BMAD story spec?
- Are the right patterns and abstractions used?
- Is the code consistent with the existing codebase?

Round 2 - Robustness & Test Coverage:
- Review findings from Round 1. Were issues addressed?
- Are edge cases handled?
- Is error handling comprehensive?
- Is test coverage sufficient?

Round 3 - Security, Performance & Cleanup:
- Review findings from Rounds 1 and 2. Were issues addressed?
- Any security concerns (input validation, auth, injection)?
- Performance bottlenecks?
- Dead code, TODOs, formatting?
- Final polish

### After Each Review Round
Update review-tracker.yaml:
- Increment 'completed' count
- Set 'last_round_focus' to the focus area
- Append new findings to 'findings' list
- If the review found issues that need code changes:
  Set 'needs_fix: true' and describe in 'fix_required'
  Do NOT increment to next round yet
  Instead: apply the fix, then re-run the SAME round
  Max 2 fix attempts per round (then move on with a note)

### Review Tracker Template

If .bmad-output/review-tracker.yaml does not exist, create it:
```yaml
# Auto-generated by bmad-sprint
review_rounds: {}
```

For each story entering review, add an entry:
```yaml
review_rounds:
  <story-key>:
    completed: 0
    max_rounds: 3
    rounds:
      - round: 1
        focus: 'architecture and spec conformance'
        status: pending
        findings: []
        fix_attempts: 0
      - round: 2
        focus: 'robustness and test coverage'
        status: pending
        findings: []
        fix_attempts: 0
      - round: 3
        focus: 'security, performance, and cleanup'
        status: pending
        findings: []
        fix_attempts: 0
```

## Safety Rules
- Do NOT ask any questions. Work fully autonomously.
- If stuck on a story after 3 attempts, set status to 'blocked' 
  and add a note in review-tracker.yaml, then move to the next story.
- Never output the completion promise prematurely.
- Process ONE story phase per iteration, then let the loop restart.

## Completion
If ALL stories in the sprint have status 'done' or 'blocked':
Output <promise>SPRINT_COMPLETE</promise>
